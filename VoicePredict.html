SISTEM IDENTIFIKASI SUARA UNTUK KLASIFIKASI AKTIVITAS BUKA DAN TUTUP#

Proyek ini dibuat untuk mengklasifikasikan suara manusia ke dalam kategori tertentu ‚Äî yaitu suara ‚Äúbuka‚Äù dan ‚Äútutup‚Äù. Sistem dikembangkan menggunakan bahasa pemrograman Python dengan memanfaatkan beberapa pustaka utama: Streamlit untuk membangun antarmuka web, Librosa untuk mengekstraksi fitur dari sinyal audio, dan Scikit-learn untuk melakukan pelatihan serta prediksi menggunakan algoritma Random Forest Classifier.

Alur kerja aplikasi dimulai dari pemrosesan dataset suara yang disimpan dalam format CSV, dilanjutkan dengan ekstraksi fitur audio, pelatihan model klasifikasi, penyimpanan model hasil pelatihan, hingga tahap akhir berupa peluncuran aplikasi berbasis web yang mampu mengenali suara secara langsung (real-time).

Proses Pelatihan Model (train_model.py)#

Pada tahap awal, dilakukan pelatihan model (training) melalui file train_model.py. Proses ini dimulai dengan memuat dataset suara yang telah diekstraksi sebelumnya, yaitu bukatutup.csv, yang berisi fitur-fitur hasil ekstraksi audio beserta label kelasnya. Dataset dibaca menggunakan library pandas sehingga dapat diolah dalam bentuk DataFrame.

Fitur-fitur numerik yang menggambarkan karakteristik suara disimpan dalam variabel X, sedangkan label kategori suara (misalnya ‚Äúbuka‚Äù atau ‚Äútutup‚Äù) disimpan dalam variabel y. Label teks ini kemudian dikonversi menjadi format numerik menggunakan LabelEncoder, agar bisa diproses oleh algoritma pembelajaran mesin.

Selanjutnya, data dibagi menjadi data latih (train) dan data uji (test) menggunakan fungsi train_test_split dengan proporsi 80% untuk pelatihan dan 20% untuk pengujian. Pembagian ini bertujuan untuk mengevaluasi kemampuan model dalam mengenali data baru yang sebelumnya belum pernah dilihat.

Model yang digunakan adalah Random Forest Classifier, yaitu metode ensemble learning yang menggabungkan beberapa decision tree untuk menghasilkan prediksi akhir yang lebih akurat dan stabil. Pada kode, model dibuat dengan parameter n_estimators=150, artinya proses pelatihan melibatkan 150 pohon keputusan.

model = RandomForestClassifier(n_estimators=150, random_state=42)
model.fit(X_train, y_train)

Setelah model selesai dilatih, tahap berikutnya adalah evaluasi menggunakan data uji untuk menilai seberapa akurat model tersebut. Tingkat akurasi dihitung dengan fungsi accuracy_score, sementara performa tiap kelas ditampilkan melalui classification_report. Apabila akurasi yang diperoleh sudah memadai, model beserta label encoder disimpan ke dalam folder models menggunakan Joblib, sehingga bisa dipakai kembali saat proses prediksi.

# Simpan model dan label encoder
os.makedirs("models", exist_ok=True)
joblib.dump(model, "models/classifier.pkl")
joblib.dump(le, "models/label_encoder.pkl")

Dengan begitu, file train_model.py menghasilkan dua berkas utama: classifier.pkl, yang menyimpan model Random Forest yang sudah dilatih, dan label_encoder.pkl, yang menyimpan pengkodean label. Kedua berkas ini nantinya digunakan oleh sistem utama untuk proses identifikasi suara.

Proses Ekstraksi Fitur Suara (preprocess.py)#

File preprocess.py berperan sebagai komponen utama dalam proses ekstraksi fitur suara dari file audio berformat .wav. Tahap ini sangat penting karena model machine learning tidak dapat langsung memahami bentuk gelombang suara mentah, melainkan membutuhkan representasi numerik berupa fitur akustik yang mencerminkan karakteristik dari suara tersebut.

Proses dimulai dengan memuat file audio menggunakan fungsi librosa.load(file_path, sr=None), yang menghasilkan dua keluaran: sinyal audio (y) dan frekuensi sampling (sr). Setelah itu, sistem menghitung berbagai fitur utama untuk merepresentasikan suara, di antaranya:

  1. Zero Crossing Rate (ZCR) ‚Äî menghitung seberapa sering sinyal melintasi titik nol, yang menggambarkan tingkat perubahan sinyal. Nilai rata-rata (zcr_mean) dan standar deviasi (zcr_std) diambil untuk memberikan informasi tentang dinamika suara.

  2. RMS Energy (Root Mean Square) ‚Äî mengukur energi atau kekuatan sinyal, menunjukkan seberapa keras suara tersebut. Dua nilai statistik yang diambil adalah rata-rata (rmse_mean) dan standar deviasi (rmse_std).

  3. Fitur Spektral ‚Äî meliputi:

  • Spectral Centroid, menunjukkan di mana ‚Äúpusat‚Äù spektrum frekuensi berada, menandakan apakah suara bernada tinggi atau rendah.

  • Spectral Bandwidth, mengukur lebar rentang frekuensi.

  • Spectral Rolloff, menunjukkan frekuensi di mana sebagian besar energi suara terkonsentrasi. Ketiga fitur ini masing-masing diambil nilai rata-rata dan standar deviasinya.

  1. MFCC (Mel-Frequency Cepstral Coefficients) ‚Äî fitur paling umum dalam pengenalan suara yang meniru cara pendengaran manusia terhadap frekuensi. Dari 13 koefisien MFCC pertama, sistem mengambil nilai rata-rata dan standar deviasi, menghasilkan total 26 nilai.

Seluruh hasil perhitungan tersebut kemudian digabungkan menjadi satu vektor fitur: sembilan fitur utama non-MFCC (ZCR, RMS, Spektral) ditambahkan dengan 26 nilai dari MFCC, sehingga membentuk vektor berdimensi 35. Hasil akhirnya diubah menjadi array 2D (reshape(1, -1)) agar dapat digunakan oleh model untuk prediksi.

Jika proses ekstraksi gagal ‚Äî misalnya karena file audio rusak, tidak sesuai format, atau tidak bisa diproses ‚Äî maka blok except akan menangani kesalahan dan menampilkan pesan error seperti: ‚ùå Error ekstraksi¬†fitur:¬†‚Ä¶

features = [
    zcr_mean, zcr_std,
    rmse_mean, rmse_std,
    centroid_mean, centroid_std,
    bandwidth_mean, bandwidth_std,
    rolloff_mean, rolloff_std
]

for mean, std in zip(mfcc_means, mfcc_stds):
    features.extend([mean, std])

Proses Prediksi Suara (predict.py)#

Tahap selanjutnya adalah proses prediksi suara, yang dijalankan melalui file predict.py. Pada tahap ini, model yang sebelumnya telah dilatih dimuat kembali dari folder models menggunakan fungsi joblib.load(). Fungsi predict_voice(features) kemudian menerima masukan berupa fitur suara yang dihasilkan dari proses ekstraksi di file preprocess.py, dan mengembalikan dua output utama: kategori suara yang terdeteksi serta tingkat kepercayaan model terhadap prediksi tersebut.

Prediksi dilakukan dengan memanggil fungsi model.predict(features), sedangkan tingkat kepercayaan dihitung berdasarkan probabilitas tertinggi dari keluaran model.predict_proba(features). Nilai kepercayaan tersebut kemudian dikonversi ke dalam bentuk persentase agar lebih mudah dipahami.

Dengan pendekatan ini, sistem tidak hanya menampilkan hasil klasifikasi seperti ‚Äúbuka‚Äù atau ‚Äútutup‚Äù, tetapi juga memberikan informasi tentang seberapa besar keyakinan model terhadap hasil yang dihasilkan.

pred = model.predict(features)
prob = model.predict_proba(features)
class_name = label_encoder.inverse_transform(pred)[0]
confidence = np.max(prob) * 100

Aplikasi Antarmuka Streamlit (app.py)#

File app.py berfungsi sebagai antarmuka pengguna berbasis web yang dibuat menggunakan framework Streamlit. Melalui aplikasi ini, pengguna dapat melakukan pengujian suara secara langsung tanpa perlu menulis kode Python secara manual.

Saat aplikasi dijalankan, pengguna akan disuguhkan tampilan dengan judul ‚ÄúVoice Identification App‚Äù serta opsi untuk mengunggah file audio berformat .wav. Setelah file berhasil diunggah, sistem akan menampilkan audio tersebut agar dapat diputar kembali, lalu menunggu pengguna menekan tombol ‚ÄúüëÄ Identifikasi Suara‚Äù untuk memulai proses analisis suara.

import streamlit as st
import os
from utils.preprocess import extract_features
from utils.predict import predict_voice

st.set_page_config(page_title="Voice Identification App", page_icon="üé§")

st.title("üéß Voice Identification App")
st.write("Unggah file audio (format .wav) untuk dikenali suaranya.")

uploaded_file = st.file_uploader("Pilih file audio (.wav)", type=["wav"])

if uploaded_file is not None:
    with open("temp.wav", "wb") as f:
        f.write(uploaded_file.getbuffer())

    st.audio(uploaded_file, format="audio/wav")

    if st.button("üëÄ Identifikasi Suara"):
        with st.spinner("Sedang memproses..."):
            features = extract_features("temp.wav")
            if features is not None:
                pred_class, confidence = predict_voice(features)
                if pred_class:
                    st.success(f"**Hasil Prediksi:** {pred_class}")
                    st.info(f"Kepercayaan: {confidence:.2f}%")
                else:
                    st.error("Gagal melakukan prediksi.")
            else:
                st.error("Gagal mengekstraksi fitur dari audio.")
else:
    st.info("Silakan unggah file audio terlebih dahulu.")

uploaded_file = st.file_uploader("Pilih file audio (.wav)", type=["wav"])

Saat tombol ditekan, aplikasi akan menjalankan fungsi extract_features() dari file preprocess.py untuk mengekstraksi ciri khas dari suara yang diunggah. Hasil ekstraksi tersebut kemudian dikirim ke fungsi predict_voice() dalam file predict.py guna memperoleh hasil prediksi. Jika proses berjalan sukses, sistem akan menampilkan kategori suara beserta tingkat kepercayaan model melalui antarmuka yang interaktif dan mudah dipahami pengguna.

if st.button("üëÄ Identifikasi Suara"):
    with st.spinner("Sedang memproses..."):
        features = extract_features("temp.wav")
        if features is not None:
            pred_class, confidence = predict_voice(features)
            if pred_class:
                st.success(f"**Hasil Prediksi:** {pred_class}")
                st.info(f"Kepercayaan: {confidence:.2f}%")

Dengan demikian, pengguna cukup mengunggah file audio, dan sistem akan secara otomatis mengenali apakah suara tersebut termasuk kategori ‚Äúbuka‚Äù atau ‚Äútutup‚Äù.
